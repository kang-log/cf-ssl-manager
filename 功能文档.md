# Cloudflare 域名证书自动化申请工具 — 功能文档

## 1. 项目概述

### 1.1 项目名称
CF SSL Cert Manager（CF 证书管理器）

### 1.2 项目目标
基于 Cloudflare DNS 验证方式，提供图形化界面，实现 SSL/TLS 证书的自动化申请与下载。用户无需手动操作 DNS 记录，程序全程自动完成域名验证和证书签发。

### 1.3 目标用户
- 拥有 Cloudflare 托管域名的个人开发者或运维人员
- 需要为多个域名/子域名批量申请证书的管理员

### 1.4 技术选型
- **目标平台**：Windows 10/11（64 位）
- **GUI 框架**：Wails v2 + Vue3 + Element Plus
- **后端核心**：Go 1.22+
- **ACME 客户端**：`go-acme/lego` v4（内置 Let's Encrypt / Cloudflare DNS provider）
- **Cloudflare API**：`cloudflare/cloudflare-go`（官方 Go SDK）
- **证书解析**：Go 标准库 `crypto/x509`
- **数据存储**：GORM + SQLite（`gorm.io/driver/sqlite`）
- **打包发布**：`wails build` → 单个 .exe 安装包（约 20MB）

## 3. 功能模块详细说明

### 3.1 Cloudflare 账户配置模块

#### 3.1.1 功能描述
用户首次使用时输入 Cloudflare 凭证信息，程序验证后保存以供后续使用。

#### 3.1.2 输入字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 邮箱（Email） | text | 是 | Cloudflare 账户邮箱 |
| API Key | password | 是 | Cloudflare Global API Key |
| API Token | password | 否 | Cloudflare API Token（推荐，权限更精细） |

#### 3.1.3 验证逻辑
- 调用 `GET https://api.cloudflare.com/client/v4/user` 验证凭证有效性
- 若使用 API Token，调用 `GET https://api.cloudflare.com/client/v4/user/tokens/verify`
- 验证通过后，展示账户下的域名列表
- 凭证信息使用 AES-256 加密后存储于本地配置文件

#### 3.1.4 配置文件路径
- Windows: `%APPDATA%/cf-ssl-manager/config.json`

---

### 3.2 域名选择模块

#### 3.2.1 功能描述
验证通过后，自动拉取 Cloudflare 账户下所有托管域名，用户可选择目标域名。

#### 3.2.2 域名列表
- 调用 `GET https://api.cloudflare.com/client/v4/zones` 获取域名列表
- 展示信息：域名名称、状态（Active/Pending）、Plan 类型、Nameserver
- 支持搜索/筛选功能

#### 3.2.3 证书域名输入
用户在选定域名后，填写需要申请证书的具体域名，支持以下格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| 单域名 | `example.com` | 仅主域名 |
| 子域名 | `www.example.com` | 单个子域名 |
| 泛域名 | `*.example.com` | 通配符，匹配所有一级子域名 |
| 多域名（SAN） | `example.com, *.example.com, www.example.com` | 多个 Subject Alternative Name |
| 多子域名 | `api.example.com, cdn.example.com` | 多个子域名 |

> **注意**：泛域名 `*.example.com` 仅匹配一级子域名（如 `www.example.com`），不匹配多级子域名（如 `a.b.example.com`）。如需多级通配符，需单独申请 `*.sub.example.com`。

#### 3.2.4 校验规则
- 每个输入域名必须属于当前 Cloudflare 账户下已托管的根域名
- 域名格式校验（RFC 1035 标准）
- 泛域名仅允许在最左侧标签使用 `*`
- SAN 数量限制：最多 100 个（CA 限制）

### 3.3 证书品牌选择模块

#### 3.3.1 支持的 CA

| 品牌 | ACME Directory URL | 证书有效期 | 特点 |
|------|-------------------|-----------|------|
| Let's Encrypt | `https://acme-v02.api.letsencrypt.org/directory` | 90 天 | 免费、社区驱动、最广泛使用 |
| LiteSSL | `https://acme.litessl.com/directory` | 90 天 | 商业 ACME CA、免费基础证书 |

#### 3.3.2 申请参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 证书品牌 | select | Let's Encrypt | 选择 CA |
| 密钥算法 | select | ECDSA P-256 | 可选 RSA 2048/4096 或 ECDSA P-256/P-384 |
| 证书有效期 | readonly | 90 天 | 由 CA 决定 |
| 是否包含证书链 | checkbox | true | 是否下载完整证书链 |

#### 3.3.3 Let's Encrypt 环境

| 环境 | URL | 用途 |
|------|-----|------|
| Production | `https://acme-v02.api.letsencrypt.org/directory` | 正式证书 |
| Staging | `https://acme-staging-v02.api.letsencrypt.org/directory` | 测试用，不受速率限制 |

### 3.4 ACME 证书申请引擎（核心模块）

#### 3.4.1 申请流程

```
用户点击"申请证书"
        │
        ▼
① 创建 ACME Account（首次）
   POST /acme/new-acct
   - 同意 Terms of Service
   - 保存 Account Key
        │
        ▼
② 创建 Order（证书订单）
   POST /acme/new-order
   - identifiers: [{type: "dns", value: "example.com"}, ...]
        │
        ▼
③ 获取 Authorization（域名授权）
   GET /acme/authz/{id}
   - 返回 DNS-01 challenge
        │
        ▼
④ 自动创建 DNS TXT 记录（CF API）
   POST /zones/{zone_id}/dns_records
   - type: "TXT"
   - name: "_acme-challenge.example.com"
   - content: challenge token
        │
        ▼
⑤ 等待 DNS 传播
   - 轮询检查 TXT 记录是否生效
   - 最大等待时间：5 分钟
   - 间隔：10 秒
        │
        ▼
⑥ 通知 ACME Server 验证
   POST /acme/challenge/{token}
        │
        ▼
⑦ 轮询 Order 状态
   GET /acme/order/{order_url}
   - 状态：pending → ready → valid
        │
        ▼
⑧ 生成 CSR 并提交
   - 生成私钥
   - 生成 CSR（Certificate Signing Request）
   POST /acme/order/{id}/finalize
        │
        ▼
⑨ 下载证书
   GET /acme/cert/{cert_url}
   - 保存证书文件和私钥
        │
        ▼
⑩ 清理 DNS 记录
   DELETE /zones/{zone_id}/dns_records/{id}
```

#### 3.4.2 Cloudflare DNS 自动验证详细逻辑

**创建 TXT 记录：**
```
POST https://api.cloudflare.com/client/v4/zones/{zone_id}/dns_records

Headers:
  X-Auth-Email: {user_email}
  X-Auth-Key: {api_key}
  Content-Type: application/json

Body:
{
  "type": "TXT",
  "name": "_acme-challenge.{domain}",
  "content": "{dns_challenge_token}",
  "ttl": 120
}
```

**多域名/泛域名处理：**
- 每个域名（SAN）需要独立的 `_acme-challenge` TXT 记录
- 泛域名 `*.example.com` 的验证记录为 `_acme-challenge.example.com`（与主域名相同）
- 若同时申请 `example.com` 和 `*.example.com`，共用一条 TXT 记录

**记录传播检查：**
```
使用 DNS 查询验证 TXT 记录已生效：
- 通过 Cloudflare API 查询已创建的记录
- 或通过公共 DNS（如 1.1.1.1）查询 TXT 记录
- 确认记录值与 challenge token 一致
```

**记录清理：**
- 证书申请成功后，自动删除所有创建的 `_acme-challenge` TXT 记录
- 申请失败时也进行清理
- 记录 ID 保存在内存中，确保精准删除

#### 3.4.3 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| CF API 认证失败 | 提示检查凭证，终止流程 |
| 域名未托管在 CF | 提示用户将域名添加到 CF |
| DNS 记录创建失败 | 重试 3 次，失败后提示并清理 |
| DNS 传播超时 | 提示网络问题，提供手动验证选项 |
| ACME 验证失败 | 展示具体错误原因，提供重试 |
| 速率限制（Rate Limit） | 提示等待时间，建议切换 Staging 环境测试 |
| CSR 生成失败 | 提示检查域名格式 |

### 3.5 证书下载与管理模块

#### 3.5.1 输出文件

证书申请成功后，生成以下文件：

| 文件名 | 内容 |
|--------|------|
| `{domain}.crt` | 域名证书（PEM 格式） |
| `{domain}.key` | 私钥文件（PEM 格式） |
| `{domain}.chain.crt` | 完整证书链（含中间证书） |
| `{domain}.fullchain.pem` | 证书 + 证书链合并文件 |
| `ca.crt` | CA 根证书 |

#### 3.5.2 证书存储路径
- 默认路径：`%USERPROFILE%/cf-ssl-certs/{domain}/`
- 支持用户自定义存储路径
- 文件权限：私钥文件通过 NTFS ACL 限制为仅当前用户可读写

#### 3.5.3 证书列表

所有申请过的证书以列表形式展示，方便用户快速浏览和管理历史证书。

**列表字段：**

| 列名 | 说明 | 示例 |
|------|------|------|
| 域名 | 证书绑定的主域名 | `example.com` |
| SAN 数量 | 包含的域名总数（含主域名） | `3` |
| 证书品牌 | 签发 CA | Let's Encrypt / LiteSSL |
| 密钥算法 | 证书使用的密钥类型 | ECDSA P-256 / RSA 2048 |
| 申请时间 | 证书签发时间 | `2026-04-26 14:30:00` |
| 到期时间 | 证书过期时间 | `2026-07-25 14:30:00` |
| 剩余天数 | 距到期剩余天数（实时计算） | `90 天` |
| 状态 | 证书当前状态（颜色标签） | 🟢 有效 / 🟡 即将过期 / 🔴 已过期 |

**列表交互：**
- 支持按域名、品牌、状态进行搜索和筛选
- 支持按申请时间、到期时间、剩余天数排序（升序/降序）
- 点击行展开详情面板，显示 SAN 列表和文件路径
- 双击行或点击"查看详情"跳转至证书查看页（见 3.7 证书查看模块）
- 支持多选批量操作（批量删除、批量导出）

**列表操作按钮：**

| 操作 | 说明 |
|------|------|
| 查看详情 | 跳转证书查看页，展示完整证书信息 |
| 下载文件 | 打开文件存储目录或导出证书文件 |
| 续期证书 | 使用相同参数重新申请证书 |
| 删除记录 | 删除本地记录（可选同时删除证书文件） |
| 复制域名 | 一键复制主域名到剪贴板 |

**列表样式示例：**

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
```

**数据存储：**
- 本地数据库（SQLite）记录所有申请历史

- 主表字段：id、domain、san_list、ca_brand、key_algo、issued_at、expires_at、file_path、status、created_at
- 每次启动时自动刷新剩余天数和状态

### 3.6 证书到期提醒模块

#### 3.6.1 功能描述
监控已申请证书的到期时间，提前提醒用户续期。

#### 3.6.2 提醒规则

| 提醒时间 | 提醒级别 | 说明 |
|---------|---------|------|
| 到期前 30 天 | info | 建议安排续期 |
| 到期前 14 天 | warning | 需要尽快续期 |
| 到期前 7 天 | urgent | 紧急，即将到期 |
| 到期前 1 天 | critical | 临界，必须立即续期 |

#### 3.6.3 自动续期（可选）
- 支持配置自动续期开关
- 自动续期在到期前 30 天执行
- 续期流程与新申请一致
- 续期失败时发送通知

### 3.7 证书查看模块

#### 3.7.1 功能描述
支持对已申请或外部导入的证书进行详细查看，解析证书内容并以结构化方式展示，方便用户确认证书信息、排查问题。

#### 3.7.2 证书基本信息展示

| 字段 | 说明 | 示例 |
|------|------|------|
| 主题（Subject） | 证书绑定的主体信息 | `CN=example.com` |
| 颁发者（Issuer） | 签发证书的 CA | `CN=R3, O=Let's Encrypt, C=US` |
| 序列号（Serial Number） | CA 分配的唯一编号 | `04:AB:C3:...` |
| 签名算法 | 证书使用的签名算法 | `SHA256-RSA` / `SHA256-ECDSA` |
| 公钥算法 | 证书公钥的算法与长度 | `RSA 2048` / `ECDSA P-256` |
| 有效期起始（Not Before） | 证书生效时间 | `2026-04-26 00:00:00 UTC` |
| 有效期截止（Not After） | 证书过期时间 | `2026-07-25 00:00:00 UTC` |
| 剩余天数 | 距到期剩余天数（动态计算） | `90 天` |

#### 3.7.3 SAN（主题备用名称）展示
- 列出证书中包含的所有 SAN 条目
- 区分类型：DNS 名称、IP 地址、泛域名
- 以列表形式展示，支持复制单条

```
SAN 列表：
  ● example.com          (DNS)
  ● *.example.com        (DNS - 泛域名)
  ● www.example.com      (DNS)
  ● 192.168.1.1          (IP)
```

#### 3.7.4 证书链查看
- 展示完整的证书信任链（从当前证书到根证书）
- 每一级证书可展开查看详情
- 标注每一级的验证状态（受信任 / 不受信任 / 过期）

```
证书信任链：
  ✅ 根证书：ISRG Root X1 (Let's Encrypt)
     └── ✅ 中间证书：R3
            └── ✅ 域名证书：example.com
```

#### 3.7.5 指纹信息

| 算法 | 用途 |
|------|------|
| SHA-1 | 传统标识，浏览器兼容性检查 |
| SHA-256 | 现代标准指纹，推荐使用 |

- 支持一键复制指纹值

#### 3.7.6 PEM 内容查看
- 以代码块形式展示证书的原始 PEM 文本内容
- 支持切换查看：域名证书 / 证书链 / 完整证书链（fullchain） / 私钥（脱敏显示）
- 私钥查看默认隐藏内容，需用户手动点击"显示"确认
- 支持一键复制 PEM 内容

```
┌─ 证书内容 ──────────────────────────────────────┐
│ [域名证书] [证书链] [完整证书链] [私钥]           │
│                                                   │
│ ┌─────────────────────────────────────────────┐  │
│ │ -----BEGIN CERTIFICATE-----                 │  │
│ │ MIIFazCCBFOgAwIBAgISA...                    │  │
│ │ ...                                         │  │
│ │ -----END CERTIFICATE-----                   │  │
│ └─────────────────────────────────────────────┘  │
│                                    [复制] [导出]  │
└───────────────────────────────────────────────────┘
```

#### 3.7.7 证书状态指示
根据证书当前状态，以不同颜色标签直观展示：

| 状态 | 颜色 | 条件 | 说明 |
|------|------|------|------|
| 有效 | 🟢 绿色 | 当前时间在有效期内 | 证书正常可用 |
| 即将过期 | 🟡 黄色 | 剩余 ≤ 30 天 | 建议尽快续期 |
| 已过期 | 🔴 红色 | 当前时间超过 Not After | 证书不可用 |
| 未生效 | ⚪ 灰色 | 当前时间早于 Not Before | 证书尚未生效 |

#### 3.7.8 外部证书导入查看
- 支持导入本地 `.crt`、`.pem`、`.cer`、`.p12`、`.pfx` 格式的证书文件进行查看
- 拖拽文件到证书查看窗口即可解析
- 导入的证书仅用于查看，不会自动保存到证书库（用户可选择保存）

#### 3.7.9 在线证书检测
- 支持输入域名，通过 HTTPS 连接直接获取远程服务器的证书信息
- 展示远程证书详情，对比本地证书是否一致
- 适用于验证证书部署是否正确

```
输入域名：example.com
端口：443
[检测]

检测结果：
  证书主题：CN=example.com
  颁发者：R3 (Let's Encrypt)
  有效期：2026-04-26 ~ 2026-07-25 (剩余 90 天)
  指纹：SHA-256: AA:BB:CC:...
  与本地证书：✅ 一致
```

## 5. 安全设计

### 5.1 凭证安全
- Cloudflare API Key 使用 AES-256-GCM 加密存储
- 加密密钥基于用户系统信息派生（PBKDF2）
- 内存中使用后及时清理敏感数据

### 5.2 私钥安全
- 生成的私钥文件通过 NTFS ACL 设置严格的访问权限（仅当前用户可读写）
- 支持私钥加密（可选密码保护）
- 不将私钥上传到任何外部服务

### 5.3 网络安全
- 所有 API 请求使用 HTTPS
- 验证 SSL 证书有效性
- 支持代理配置（HTTP/SOCKS5）

## 9. 术语表

| 术语 | 说明 |
|------|------|
| ACME | Automatic Certificate Management Environment，自动证书管理环境 |
| CSR | Certificate Signing Request，证书签名请求 |
| SAN | Subject Alternative Name，证书的主体备用名称 |
| DNS-01 Challenge | ACME 的一种域名验证方式，通过创建 DNS TXT 记录验证域名所有权 |
| 泛域名 | Wildcard Domain，使用 `*` 通配符匹配多个子域名的证书 |
| PEM | Privacy Enhanced Mail，一种 Base64 编码的证书/密钥格式 |
| Subject | 证书主体，即证书绑定的域名/组织信息 |
| Issuer | 证书颁发者，即签发该证书的 CA |
| SAN | Subject Alternative Name，证书中可包含多个域名/IP |
| 指纹（Fingerprint） | 证书内容的哈希值，用于唯一标识证书 |
| PKCS#12 | 一种证书打包格式（.p12/.pfx），包含证书和私钥 |
